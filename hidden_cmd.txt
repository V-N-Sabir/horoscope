Когда вы запускаете исполняемый файл с помощью команды `start /min`, консоль всё равно может быть видна, только она свёрнута. Для того чтобы скрыть консольное окно, есть несколько подходов, в зависимости от того, что вам нужно.

### Способ 1: Использование VBScript для скрытия консольного окна

Вместо того чтобы запускать `.exe` напрямую с помощью BAT-файла, вы можете использовать **VBScript** для запуска программы в скрытом режиме. Вот как это можно сделать:

1. Создайте новый файл с расширением `.vbs`, например, `run_hidden.vbs`.
2. Вставьте в файл следующий код:

   ```vbscript
   Set objShell = CreateObject("WScript.Shell")
   objShell.Run "1cenv.exe", 0, False
   ```

   - `0` — означает скрытый режим (консоль не будет видна).
   - `False` — указывает, что выполнение скрипта не будет ожидать завершения программы.

3. В вашем BAT-файле, вместо вызова `1cenv.exe`, вызовите `run_hidden.vbs`:

   ```batch
   start run_hidden.vbs
   ```

Теперь при запуске вашего BAT-файла консольное окно не будет видно.

### Способ 2: Использование `nssm` (Non-Sucking Service Manager)

Если вы хотите, чтобы приложение вообще работало без консольного окна, можно использовать инструмент **nssm** для превращения вашего исполняемого файла в сервис Windows. Это более сложный способ, но полезен, если нужно запускать программу в фоновом режиме постоянно.

- Скачайте **nssm** [с официального сайта](https://nssm.cc/download).
- Используйте команду для установки сервиса:

  ```batch
  nssm install MyService "C:\path\to\1cenv.exe"
  ```

Этот способ будет запускать программу как сервис, скрывая её консольное окно.
